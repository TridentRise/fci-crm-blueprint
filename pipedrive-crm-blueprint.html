Firm Hierarchy & Relationship Mapping

Financial services firms operate in layered hierarchies — an enterprise broker-dealer has a home office, regional offices, and dozens of branch offices. An insurance carrier has managing general agents, agencies, and sub-agencies. Pipedrive doesn’t natively model parent–child org relationships, so we build it with custom fields and linked orgs. This is critical for FCI because a single enterprise deal can cascade into 10–50 branch deployments.

Field NameTypeOptions / Notes
Org Level Dropdown Enterprise / Parent • Home Office / HQ • Regional Office • Branch Office • Agency • Sub-Agency • Independent / Standalone
Parent Organization Text Exact name of the parent org in Pipedrive. For branch offices, this is the broker-dealer or carrier. For agencies, this is the carrier or MGA. Must match exactly for filtering.
Enterprise Group Text Top-level enterprise name that ties all related orgs together. Example: all Cetera branches, LPL offices, or Raymond James offices share the same Enterprise Group value.
Affiliate Relationship Dropdown Affiliated (captive to parent) • Independent (uses parent for clearing/compliance only) • Hybrid • N/A
Referral Direction Dropdown Top-Down (enterprise mandated to branches) • Bottom-Up (branch discovered, expanding to enterprise) • Lateral (peer referral between branches) • None
Branch Count (Enterprise Only) Number Total number of branches / offices under this enterprise org. Populated only on the parent-level record.
Branches Deployed Number Number of branches currently running FCI services. Used to track rollout progress against Branch Count.
Home Office Contact Text Name of the primary compliance or IT contact at the home office level — the person who can authorize enterprise-wide mandates.
Enterprise Deal Link Text URL or name of the master enterprise deal in the New Business pipeline. Links branch-level orgs back to the parent deal for reporting.
How to model hierarchies in Pipedrive. Create one org record per entity (enterprise, home office, each branch). Populate the Parent Organization and Enterprise Group fields consistently — these are your join keys. Use Pipedrive’s “Related Organizations” feature to create formal links between parent and child orgs. Then build saved filters like “All orgs where Enterprise Group = Cetera” to see the full family tree. When an enterprise deal closes, create expansion deals in Pipeline 3 for each branch rollout.
Common Hierarchy Patterns
PatternExampleHow to Map
Broker-Dealer → Branches LPL Financial → 50+ independent branch offices Enterprise Group = "LPL Financial" on all orgs. Parent = LPL on each branch. Org Level = Enterprise on LPL, Branch Office on each office.
Carrier → MGA → Agencies Nationwide → Regional MGA → Local agencies Enterprise Group = "Nationwide" on all. MGA parent = Nationwide. Agency parent = MGA name. Three-tier hierarchy tracked via Parent Organization field at each level.
Home Office → Regional → Local Cetera HQ → Cetera Northeast → Individual offices Enterprise Group = "Cetera" on all. Parent Organization cascades down each level. Org Level distinguishes Home Office, Regional Office, Branch Office.
Independent RIA (No Hierarchy) Single-office wealth management firm Org Level = Independent / Standalone. Parent Organization and Enterprise Group left blank. Standard single-org tracking.

08 — Leads Inbox
Leads vs. Deals
Leads Inbox Strategy

Pipedrive's Leads Inbox is the front door. Every new name enters as a Lead — not a Deal. A Lead becomes a Deal only when it passes explicit conversion criteria. This keeps your pipelines clean, your forecasts honest, and prevents unqualified noise from inflating pipeline value.

What Is a Lead vs. a Deal?
Lead (Leads Inbox)Deal (Pipeline)
Unqualified contact — interest exists but nothing confirmedQualified opportunity — real need, right firm type, authority contact engaged
No pipeline stage, no forecast weightSits in a pipeline stage, counts toward forecast
Goal: qualify or disqualify quicklyGoal: advance through stages toward close
Owned by whoever is doing initial outreachOwned by the rep running the opportunity
Example: someone books a Calendly intro callExample: after the call, they want a gap analysis scheduled

What Doesn’t Convert

Not every lead becomes a deal. Leads that fail conversion criteria should be handled, not ignored:

SituationAction
Wrong firm type (not in FCI's market)Archive the lead. Add a note explaining why.
Right firm, but no need or interest after initial contactArchive with note. Add to nurture list — revisit in 6 months.
Can't reach anyone / no response after 3 attemptsArchive the lead. Log final attempt date.

The Leads Inbox is a processing queue, not a parking lot. Every lead should be touched within 24 hours of creation and either converted, archived, or scheduled for follow-up within 7 days. If a lead sits untouched for 14+ days, it's a problem — the automation section includes alerts for this.

11 — Data Hygiene
Cleanup Protocol
Data Hygiene & Dead Deal Cleanup

Before implementing any of the above, the existing Pipedrive data needs to be cleaned. Stale data destroys pipeline accuracy and makes forecasting meaningless. Here is the cleanup protocol.

Phase 1 — Dead Deal Purge
  1. Pull all open deals. Export every deal currently in an "open" state across all pipelines.
  2. Flag stale deals. Any deal with no logged activity (email, call, note, meeting) in 90+ days is dead. Mark them Lost with reason "No Decision Made" or "Timing Not Right."
  3. Review 30–90 day deals. Any deal with 30–90 days of inactivity: the owner must either log a current status note and schedule a next step, or mark it Lost.
  4. Kill zombie pipeline. Deals that have been in the same stage for 6+ months with no movement — mark them Lost immediately. If the relationship is still warm, create a fresh deal when something real happens.
  5. Validate deal values. Any deal without an MRR or ACV — either populate it or mark the deal as requiring review.
Phase 2 — Contact & Org Cleanup
  1. Deduplicate organizations — with caution. Pipedrive flagged 746 potential duplicate org groups. However, many apparent duplicates are legitimate separate offices of the same firm (e.g., Thrivent Financial with 6 records representing different branch offices, each with their own contacts and deals). In financial services, a single firm name often maps to multiple offices, regions, or broker-dealer branches. Do not bulk-merge. Only merge when two records genuinely represent the same office with the same contacts entered twice. When in doubt, keep them separate — merging destroys the office-level distinction.
  2. Deduplicate contacts. Merge duplicate person records (common: same person entered from different sources or imports). Review carefully — the same person name at different offices is not a duplicate.
  3. Classify orgs as you work them. With 9,700+ organizations, batch-classifying Firm Type, Market Segment, and Relationship Type for every record is impractical and low-value for dormant accounts. Instead, populate these fields as part of the natural workflow when actively working an account. For any deal entering the pipeline, the linked org must have Firm Type and Relationship Type populated — the new deal checklist enforces this going forward.
  4. Tag former clients. Any org that was once a client but is no longer active should have Relationship Type = Former Client.
  5. Archive irrelevant orgs. Vendors, personal contacts, and non-target firms that clutter searches — either archive or tag clearly.
Phase 3 — Ongoing Hygiene Rules
  1. Weekly pipeline review. Every Monday: review all deals, update Next Step, flag stale deals.
  2. 30-day inactivity auto-flag. Set up Pipedrive automation to label deals with 30+ days of inactivity as "Stalled."
  3. Quarterly purge. Once per quarter, run the dead deal purge again. It is cheaper to lose a stale deal from the pipeline and reopen it later than to carry false pipeline all year.
  4. New deal checklist. No deal enters the pipeline without: Contact Source, Firm Type, Assigned To, and at least one contact with Buyer Role populated.
The goal of cleanup is not to have fewer deals. It is to have accurate deals. A 15-deal pipeline where every deal is real and has a next step is infinitely more useful than a 60-deal pipeline where 40 of them are ghosts. Clean pipeline = accurate forecast = better decisions.

Insights & Dashboards
Dashboards & Reporting

Four focused dashboards in Pipedrive Insights, each tied to a clear cadence and audience. Consolidated from six to four to ensure every dashboard gets used — not just built.

Metrics requiring external data (CPA, endpoint counts, true MRR with churn) are deferred to a BI layer once 2–3 quarters of clean pipeline data exist.

Dashboard 1 — Pipeline & Revenue (Weekly Review)

Monday morning board — where the money sits and where deals are stuck.

  1. Open deals by stage — stacked bar, all five pipelines, segmented by pipeline
  2. Deal value by stage — same view weighted by deal value instead of count
  3. Deals won vs. lost — monthly trend, trailing 6 months
  4. Won deal value — monthly trend, trailing 12 months (revenue trajectory)
  5. Average deal age by stage — identifies bottlenecks and stalling points
  6. Stalled & at-risk deals — filtered to Stalled or At Risk labels (action list)
  7. Deals by urgency trigger — segmented by Urgency Trigger custom field
Dashboard 2 — Lead Source & Funnel (Monthly Strategy)

Where should FCI spend time and money to generate business?

  1. Deals created by source — bar chart segmented by Contact Source, count and value
  2. Won deals by source — which sources actually close?
  3. Conversion rate by source — deals won ÷ deals created, by source
  4. Average deal size by source — are referrals bigger than inbound?
  5. Leads created by source — monthly trend from Leads module (top-of-funnel volume)
  6. Lead-to-deal conversion — how many leads become real pipeline?
  7. Lead aging — leads in inbox by age bucket (0–7, 7–14, 14+ days)
Dashboard 3 — Market Segment (Quarterly Strategic)

Which segment is FCI winning in, and where’s the growth? Uses Pipeline as proxy for market segment (New Business, Channel & Referral Partners, Expansion & Renewal, Closers.io 2026).

  1. Pipeline Value by Segment — deal value (USD) by pipeline, segmented by deal status
  2. Won Deals by Segment — deal count by pipeline, filtered to won deals only
  3. Win Rate by Segment — win/loss conversion view by pipeline
Dashboard 4 — Sales Activity (Weekly Accountability)

Is the team doing the work? Rep-level activity tracking.

  1. Activities by Type — activity performance by type (calls, emails, meetings), this year
  2. Activities by User — activity performance by assigned user, this year
  3. Gap Analyses Completed — deal performance filtered by Gap Analysis Completed? = Yes, viewed by pipeline
Dashboard cadence. Pipeline & Revenue → every Monday. Lead Source & Funnel → first week of each month. Market Segment → quarterly board prep. Sales Activity → every Friday. Four dashboards, four rhythms, zero noise.
