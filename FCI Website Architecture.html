Phase 4 — Website Build

Website Architecture

Complete sitemap, tech stack, content readiness status, and Phase 3 gaps for the new FCI website. FastAPI + HTMX, Honest Code methodology, 24 pages across 7 sections.

Tech Stack

Honest Code · FastAPI + HTMX

Backend
FastAPI — route handlers at the boundary, pure functions in app/logic/
Frontend
HTMX — server-rendered HTML, fragment swaps, no client-side state store
Templates
Jinja2 — base.html shell, page templates, _fragment.html partials for HTMX
CSS
Custom properties in :root, BEM naming, one file per component, no Tailwind
Data
TypedDict over classes, pure functions, dict-lookup polymorphism
News CMS
Admin route with auth — create/edit/publish news articles and Cyber Bites via HTMX forms
Database
SQLite (news articles, Cyber Bites, contact form submissions) — SQL over caches (P09)
Hosting
TBD — Railway, Render, Fly.io, or self-hosted (needs persistent storage for SQLite + CMS)
Domain
fcicyber.com (existing) — DNS cutover from WordPress when ready

Project Structure (Honest Code)

app/
  main.py           — app creation, router inclusion, lifespan
  routes/
    pages.py       — public page routes (/, /about, /markets/...)
    news.py        — news listing + detail routes
    cyber_bites.py  — cyber bites listing + detail
    contact.py      — contact form handler
    admin.py       — CMS admin routes (auth-protected)
  logic/
    content.py      — pure functions for content retrieval/filtering
    contact.py      — pure validation/formatting
  templates/
    base.html       — shell (nav, footer, theme toggle)
    pages/          — full page templates
    fragments/      — HTMX partial responses
    admin/          — CMS templates
  static/
    css/            — component CSS files (BEM)
    images/         — logos, hero assets, team photos
  db.py            — SQLite connection (context manager, P12)
tests/
  test_pure.py      — pure assertions, zero mocks (P10)
  test_routes.py    — integration via httpx.AsyncClient

Recommended Build Order

Sprint 1 — Foundation

Goal: FastAPI app skeleton, base template, theme system, nav/footer, homepage rendering.

Set up the project structure per Honest Code. Create base.html with the nav structure, dark/light theme toggle (matching the hub design system), and footer. Render the homepage from the approved Website Copy content. Deploy to staging.

Sprint 2 — Core Pages

Goal: All "Ready" static pages live — About, What Changes, 3 Markets, 5 Solutions, Team, Careers.

These all have approved copy in the hub. Convert each to Jinja2 templates inheriting from base.html. HTMX fragments for market/solution selector interactions. This is 13 pages with content already written.

Sprint 3 — CMS + Dynamic Content

Goal: News and Cyber Bites running from SQLite, with admin interface for publishing.

Seed the database with the 40 migrated news articles and 40 Cyber Bites. Build listing pages with HTMX-powered filters and pagination. Build admin routes for create/edit/publish behind authentication.

Sprint 4 — Phase 3 Content + Polish

Goal: Fill the 7 content gaps, build Contact form, Speaking & Media page, Support page.

Write Markets Overview, Solutions Overview, Contact page, Support page. Polish the Question Layer into live FAQ content. Adapt the AI Messaging page. Draft Privacy Policy and Terms of Service.

Sprint 5 — Launch Prep

Goal: DNS cutover, SEO, redirects, final QA.

Set up 301 redirects from old WordPress URLs to new routes. Meta tags, Open Graph, structured data. Performance testing. Security headers. DNS cutover from WordPress to FastAPI. Monitor for 404s post-launch.
