Zero Trust 1/4: Users
Zero Trust
A Zero Trust approach considers users, endpoints, software, and networks as potential threats until proven otherwise. It starts with users. Do you know who is accessing your systems? Identity verification through multi-factor authentication, single sign-on, and password management ensures that only authorized individuals can reach private data. The question is not whether your users are trusted — it is whether their identity has been verified at the time of access.
Source: fcicyber.com/zero-trust-1-4-users
Zero Trust 2/4: Endpoints
Zero Trust
Any device used to access private data is an endpoint — and every endpoint is a potential entry point. Zero Trust requires that every device, whether corporate-owned or BYOD, has enforced cybersecurity settings, full-disk encryption, antivirus and anti-malware protection, and endpoint detection and response. If a device does not meet the security baseline, it should not access systems of private data. Managed enforcement — not user instructions — is what makes this work at scale.
Source: fcicyber.com/zero-trust-2-4-endpoints
Zero Trust 3/4: Software
Zero Trust
Zero Trust software means hardening applications and validating that users and endpoints are compliant at the time of login. This includes vendor risk management, restricted websites and software, data exfiltration protection, and web application security gateways. The question is whether the software your firm uses has been configured to enforce security — not just installed.
Source: fcicyber.com/zero-trust-3-4-software
Zero Trust 4/4: Networks
Zero Trust
Network security in a Zero Trust model means enforcing secure, encrypted communication inside and outside corporate networks. Security policies must be implemented at the network level — firewall hardening, secure remote communication via gateway VPN, and continuous monitoring. The network cannot be trusted simply because it is internal. Every connection is verified.
Source: fcicyber.com/zero-trust-4-4-networks
Managed Is Most Secure
Endpoints
For endpoint protection, you could provide instructions to your users or your IT team to manually configure settings — complex passwords, inactivity timeouts, personal firewalls, encryption. Or you could enforce them automatically, with tamper protection that prevents anyone from disabling or changing what has been configured. Managed enforcement eliminates the gap between policy and practice. When controls are enforced rather than requested, compliance is continuous — not aspirational.
Source: fcicyber.com/managed-is-most-secure
Ensure Encryption
Endpoints
Encryption protects data at rest on your device and in transit across a network. If a device is lost or stolen, full-disk encryption makes the data unreadable to anyone without the decryption key. This is not optional — it is a regulatory requirement for firms handling private data. The question is whether encryption is enforced on every device, or left to the user to enable.
Source: fcicyber.com/ensure-encryption
Removable Storage Device Encryption
Endpoints
If private data is backed up to an unencrypted external drive — a USB drive, for example — and that drive is lost or stolen, anyone can access its content. That constitutes a reportable breach. Enforced encryption of removable storage devices prevents this. The control can be applied without blocking the ability to read non-encrypted devices. The question is whether your firm enforces this or leaves it to chance.
Source: fcicyber.com/removable-storage-device-encryption
Keep Your OS Current
Endpoints
Cybersecurity regulations require a currently supported operating system. Every OS has a lifecycle defined by the manufacturer during which critical security patches are released. Once an OS reaches end-of-life, no further patches are issued — leaving every known vulnerability permanently open. If your firm has devices running unsupported operating systems, those devices are non-compliant and exposed.
Source: fcicyber.com/keep-your-os-current
Reboot Your Computer
Endpoints
One critical but simple action to improve cybersecurity and system performance: routine system reboot. Security patches, software updates, and configuration changes often require a restart to take effect. A device that has not been rebooted in weeks may be running with unpatched vulnerabilities and degraded performance. Some compliance frameworks require regular reboots as a documented practice.
Source: fcicyber.com/reboot-your-computer
Printers & Scanners
Endpoints
Printers and scanners are often forgotten endpoints — but they are important to your firm's cybersecurity posture. Some devices have features that store scanned documents or have network-accessible interfaces that can be exploited. If a printer or scanner is connected to your network and handles private data, it needs to be included in your security program.
Source: fcicyber.com/printers-and-scanners
Big Cyber Risk: Unmanaged Devices
Endpoints
One of the biggest risks in cybersecurity is allowing unmanaged devices to access private data. When a device is unmanaged, the firm has no visibility into its security posture — no enforcement of encryption, no antivirus verification, no password policy compliance. The device could be compromised and the firm would not know. Every device that touches private data must be under management.
Source: fcicyber.com/big-cyber-risk-unmanaged-devices
Password Manager Best Practices
Users & Access
When using a password manager, create complex passwords and change them regularly. Implement multi-factor authentication so every login attempt requires user approval. Configure the app properly — for example, restrict usage to specific countries. Never allow browsers to save passwords, as stored browser credentials are a known attack vector. The password manager is only as secure as its configuration.
Source: fcicyber.com/password-manager-best-practices
Multifactor Authentication
Users & Access
Multi-factor authentication (MFA) for all user accounts is a regulatory expectation and a fundamental security control. MFA verifies identity through something the user knows, something they have, and something they are — before granting access. A password alone is not sufficient. The question is whether MFA is enforced across every application that accesses private data.
Source: fcicyber.com/multifactor-authentication
Complex Usernames Reduce Risk
Users & Access
A properly created username serves as additional security. Using a complex username — rather than an obvious email address or first-name format — diminishes the risk of malicious algorithms being able to predict credentials. The username is the first half of the authentication equation. Make it count.
Source: fcicyber.com/use-of-a-complex-username-diminishes-risk-of-malicious-algorithms
Password Manager App Security
Users & Access
A password manager provides the right balance between security and time-saving access. Credentials are robustly encrypted and accessible only through your master password and MFA. The alternative — reusing passwords, writing them down, or storing them in browsers — creates the kind of vulnerability that attackers actively exploit. Protect access to private data by strengthening credentials at the source.
Source: fcicyber.com/a-password-manager-app-offers-ideal-balance-between-security-and-time-saving-access
Beware of Phishing
Users & Access
Phishing remains a leading cybersecurity threat. Attackers use increasingly sophisticated tactics to obtain private data by exploiting human behavior — posing as a known contact or trusted institution to get the recipient to click, download, or disclose. Training helps, but it is not sufficient alone. Technical controls — email filtering, link inspection, and endpoint protection — reduce the attack surface that phishing exploits.
Source: fcicyber.com/beware-of-phishing
Your Own Gateway VPN
Networks
When working remotely or traveling, the best practice for securing data across the internet is use of a business-grade firewall with a gateway VPN. This encrypts all traffic between the device and the corporate network, regardless of where the user is located. Public Wi-Fi, hotel networks, and shared connections are all potential interception points. A gateway VPN eliminates that risk.
Source: fcicyber.com/your-own-gateway-vpn
Network Security
Networks
A business-grade firewall is the first line of defense for your network. It provides advanced security features including VPN gateway to extend your secure network to remote users, intrusion prevention, content filtering, and traffic monitoring. Consumer-grade routers do not provide this level of protection. If your firm handles private data, the network must be secured at an enterprise level.
Source: fcicyber.com/network-security-tip
Travel Securely
Networks
VPN usage when traveling is the best practice for protecting data across the internet. Two major threats to be aware of: public Wi-Fi networks that can be intercepted, and untrusted networks that may redirect traffic through malicious servers. A business-grade VPN connection back to your corporate firewall ensures that all data remains encrypted in transit — regardless of the network you are connected to.
Source: fcicyber.com/travel-securely
Smartphone Security
Networks
Smartphone data is most secure when kept within your carrier's cellular network rather than free Wi-Fi. The cost savings of public Wi-Fi are not worth the security risk — these networks can be intercepted, spoofed, or used to deploy man-in-the-middle attacks. If your firm's data is accessed from a smartphone, that device and its network connection must be part of your security program.
Source: fcicyber.com/smartphone-security
Video Conferencing Best Practices
Networks
Video conferencing software is essential for remote work — but it must be configured securely. Use meeting passwords, enable waiting rooms, restrict screen sharing to the host, and disable features that allow participants to join before the host. Ensure the platform uses end-to-end encryption. The convenience of virtual meetings should not come at the cost of data exposure.
Source: fcicyber.com/video-conferencing-best-practices
Device Decommissioning
Compliance
Your firm's cybersecurity program should include a procedure for decommissioning devices that will no longer be used. Regulators request that firms formally offboard, track, and retain records of devices that accessed or stored private data. A recent SEC audit requested a list of computers decommissioned in the prior year and evidence of their proper destruction. Two options: destroy the hard disk (with photographic evidence or a third-party certificate of destruction) or securely repurpose an encrypted disk using a low-level format. Record every decommission in your cyber folder and remove the device from your asset inventory.
Source: fcicyber.com/the-importance-of-device-decommissioning
Properly Decommission a Computer
Compliance
There are two options to properly decommission a computer used to access or store private data: destroy or repurpose. You can destroy the hard disk and evidence the process, or securely repurpose an encrypted disk using the operating system's low-level format feature. Either way — document it. Regulators expect evidence of the process, not just a policy that describes it.
Source: fcicyber.com/properly-decommission-a-computer
NIST-Based Asset Inventory Report
Compliance
In the event of an audit or breach, you may be required to demonstrate a list of your assets and their cybersecurity posture. A NIST-based asset inventory report provides this — an inventory of endpoints, their security configurations, and their compliance status. This is not a nice-to-have. It is the evidence that regulators, authorities, and cyber insurers will ask for.
Source: fcicyber.com/nist-based-asset-inventory-report
POAMs (Plan of Action & Milestones)
Compliance
When reviewing cybersecurity regulation requirements, perfection is not the expectation. What regulators and authorities want to see is continual improvement — a documented plan of action with milestones (POAMs) that shows the firm is identifying gaps, prioritizing remediation, and making measurable progress. A POAM demonstrates intentionality. It transforms findings into a compliance narrative.
Source: fcicyber.com/poams
Audit Preparedness
Compliance
When it comes to audit preparedness, the question is whether your firm is a cyber ant or grasshopper. If you do not have a strong cybersecurity program and controls in place — with continuous evidence of compliance — a surprise audit or examination will expose gaps that are difficult to explain. The firms that perform best in examinations are the ones that treat compliance as an ongoing practice, not a last-minute exercise.
Source: fcicyber.com/audit-preparedness
Surprise Audit from Regulators
Compliance
Are you ready for a surprise audit from regulators? Do you have all your cybersecurity program policies in place — and can you evidence them? Would you be able to produce documentation on demand? Regulators, authorities, and cyber insurers can request evidence at any time. The firms that respond confidently are the ones with continuous evidence production, not binders assembled after the call comes in.
Source: fcicyber.com/surprise-audit-from-regulators
Vendor Risk Management
Compliance
The best way to minimize risk when working with a vendor that handles your private data is to perform vendor risk management — assessing the vendor's cybersecurity practices, certifications, and incident history before granting access to your environment. This is a regulatory expectation. If a vendor breach exposes your clients' data, the responsibility traces back to your firm.
Source: fcicyber.com/vendor-risk-management
Cybersecurity Insurance
Compliance
Having cybersecurity insurance does not guarantee payout in the event of a claim. A claim could be denied from the outset if the firm cannot demonstrate that appropriate cybersecurity controls were in place at the time of the incident. Insurers are increasingly requiring evidence of specific technical controls — MFA, endpoint protection, encryption — before approving claims. The policy is only as good as the compliance posture behind it.
Source: fcicyber.com/cybersecurity-insurance
Cost of a Cyber Breach
Compliance
The cost of a cyber breach can devastate a firm. Sixty percent of firms go out of business within six months of experiencing a breach. The damage ranges from direct financial losses and ransom payments to regulatory fines, legal fees, and reputational harm. Investment in cybersecurity protects the firm by mitigating the risk of breach and increasing the likelihood of a favorable outcome if one occurs.
Source: fcicyber.com/cost-of-a-cyber-breach
Cost vs. Benefits of Cybersecurity
Compliance
Data breaches are costly — and the investment required to prevent them is a fraction of the cost of recovery. Damage from a breach includes financial losses, regulatory fines, legal exposure, client attrition, and reputational harm. The question is not whether cybersecurity is worth the investment. It is whether the firm can afford not to make it.
Source: fcicyber.com/cost-vs-benefits-of-cybersecurity
Ransomware Protection
Users & Access
Ransomware incidents are up significantly and remain a top cyber threat against the financial sector. Implementing layered defenses — endpoint detection and response, email filtering, network segmentation, regular backups, and user awareness training — reduces the risk. But the most effective protection is managed enforcement of controls that prevent ransomware from executing in the first place.

Dark Web Data
Users & Access
Your private data or user credentials could be for sale on the dark web right now — put there by cybercriminals who work around the clock to steal and monetize compromised information. Dark web monitoring identifies when your firm's credentials appear in underground marketplaces, allowing you to respond before those credentials are used against you.

The Dark Web
Users & Access
The dark web is a hidden part of the internet used for illicit purposes. Hackers put stolen credentials there for sale — a password prefix alone can indicate that your credentials have been compromised. Monitoring the dark web for your firm's data is a proactive measure that turns a hidden threat into actionable intelligence.

Endpoints
Endpoint and network security get the most attention — but software hardening is often overlooked. When you start using an application, the default security settings are rarely sufficient. Financial services applications, CRMs, email platforms, and cloud tools all require security configuration review and hardening. The question is whether the software your firm relies on has been configured to enforce security, or left at defaults.

Proactive vulnerability scans and network penetration testing identify weaknesses before attackers do. The assessment produces documented evidence of your firm's security posture — evidence that regulators, home offices, and cyber insurers increasingly require.

Are you a seller ready to exit, or a buyer ready to acquire? A cybersecurity risk assessment should be part of every M&A transaction in financial services. For sellers, it demonstrates that the firm is secure and compliant — increasing valuation. For buyers, it identifies hidden cybersecurity liabilities before they become the acquiring firm's problem.

Hybrid Workforce Cybersecurity
Endpoints
The distinction between home, traveler, and office users is no longer operationally useful. Every user must be treated the same way — with the same level of enforcement, the same endpoint controls, the same network security. The hybrid workforce requires a security model that follows the user, not the location. Managed enforcement applied equally across every device, everywhere, is the only approach that scales.
Source: fcicyber.com/hybrid-workforce-cybersecurity
DLP Confusion
Users & Access
The confusion around "DLP" is fueled by the fact that the same acronym has two definitions: Data Leakage Prevention and Data Loss Protection. Both are important, but they address different risks. Data leakage prevention stops unauthorized transmission of data outside the firm. Data loss protection safeguards against accidental deletion or corruption. Your firm needs both — and needs to know which one is being discussed.
Source: fcicyber.com/dlp-confusion
