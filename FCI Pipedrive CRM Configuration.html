04 — New Business Pipeline
Primary Sales Pipeline
Pipeline 1 — New Business

This is FCI's primary pipeline. It follows the natural sales motion: inbound or outbound lead → gap analysis → assessment → proposal → close. The stage probabilities reflect FCI's actual conversion patterns — the gap analysis is the critical inflection point.

5%
Lead In
New lead captured. Not yet qualified.
15%
Qualified
Confirmed: right firm type, has pain, willing to talk.
30%
Gap Analysis Scheduled
30-min gap analysis on the calendar.
50%
Gap Analysis Complete
Gap analysis done. Findings delivered. Interest confirmed.
65%
Assessment / Deep Dive
Full security assessment underway or scoping discussion.
80%
Proposal Sent
Formal proposal delivered. Decision maker engaged.
90%
Verbal Commit
Verbal yes. Contract or SOW in review.
100%
Won
Signed. Handoff to onboarding.
Stage entry criteria matter. A deal should not advance to "Qualified" until you've confirmed the firm type, that they have a real pain point (upcoming exam, incident, insurance renewal, home office mandate), and that someone with authority is engaged. A deal should not advance to "Proposal Sent" without a completed gap analysis — proposals without discovery are wasted effort.

10 — Data Hygiene
Cleanup Protocol
Data Hygiene & Dead Deal Cleanup

Before implementing any of the above, the existing Pipedrive data needs to be cleaned. Stale data destroys pipeline accuracy and makes forecasting meaningless. Here is the cleanup protocol.

Phase 1 — Dead Deal Purge
  1. Pull all open deals. Export every deal currently in an "open" state across all pipelines.
  2. Flag stale deals. Any deal with no logged activity (email, call, note, meeting) in 90+ days is dead. Mark them Lost with reason "No Decision Made" or "Timing Not Right."
  3. Review 30–90 day deals. Any deal with 30–90 days of inactivity: the owner must either log a current status note and schedule a next step, or mark it Lost.
  4. Kill zombie pipeline. Deals that have been in the same stage for 6+ months with no movement — mark them Lost immediately. If the relationship is still warm, create a fresh deal when something real happens.
  5. Validate deal values. Any deal without an MRR or ACV — either populate it or mark the deal as requiring review.
Phase 2 — Contact & Org Cleanup
  1. Deduplicate organizations. Merge duplicate org records (common: same firm with slightly different names).
  2. Deduplicate contacts. Merge duplicate person records (common: same person entered from different sources).
  3. Classify every org. Every organization must have Firm Type and Relationship Type populated. Batch-update where possible.
  4. Tag former clients. Any org that was once a client but is no longer active should have Relationship Type = Former Client.
  5. Archive irrelevant orgs. Vendors, personal contacts, and non-target firms that clutter searches — either archive or tag clearly.
Phase 3 — Ongoing Hygiene Rules
  1. Weekly pipeline review. Every Monday: review all deals, update Next Step, flag stale deals.
  2. 30-day inactivity auto-flag. Set up Pipedrive automation to label deals with 30+ days of inactivity as "Stalled."
  3. Quarterly purge. Once per quarter, run the dead deal purge again. It is cheaper to lose a stale deal from the pipeline and reopen it later than to carry false pipeline all year.
  4. New deal checklist. No deal enters the pipeline without: Contact Source, Firm Type, Assigned To, and at least one contact with Buyer Role populated.
The goal of cleanup is not to have fewer deals. It is to have accurate deals. A 15-deal pipeline where every deal is real and has a next step is infinitely more useful than a 60-deal pipeline where 40 of them are ghosts. Clean pipeline = accurate forecast = better decisions.

12 — Reporting
Insights & Dashboards
Dashboards & Reporting

Build these dashboards in Pipedrive Insights so the sales team and leadership have real-time visibility into pipeline health, conversion rates, and lead source performance.

Dashboard 1 — Pipeline Health (Weekly Review)
  1. Total pipeline value (MRR and ACV) by stage
  2. Deal count by stage
  3. Average days in each stage
  4. Stale deal count (14+ days, 30+ days)
  5. Deals by urgency trigger
  6. Single-threaded deal count
Dashboard 2 — Lead Source Performance (Monthly)
  1. Deals created by lead source (count and value)
  2. Conversion rate by lead source (Lead In → Won)
  3. Average deal size by lead source
  4. Time to close by lead source
  5. Cost per acquisition by source (if marketing spend data is available)
Dashboard 3 — Market Segment View (Quarterly)
  1. Pipeline value by market segment (RIA, BD, Insurance)
  2. Win rate by market segment
  3. Average deal size by firm type
  4. Competitive win/loss by competitor type
  5. Endpoint growth (new endpoints under management this quarter)
Dashboard 4 — Sales Activity (Weekly)
  1. Activities logged by rep (calls, emails, meetings)
  2. Gap analyses scheduled and completed this week
  3. Proposals sent this week
  4. Deals advanced (moved to a later stage)
  5. Deals won and lost this week / month
Dashboard 5 — Revenue & Forecast (Monthly)
  1. Weighted pipeline value (deal value × stage probability)
  2. Commit vs. upside breakdown
  3. MRR trend (new client MRR won per month, trailing 12 months)
  4. Expansion revenue (Pipeline 3 — closed expansion deals)
  5. Client retention rate (clients renewed vs. churned)
Implementation Order
Recommended Implementation Sequence

Don't try to do all of this in one day. Here's the order that delivers the most value fastest.

  1. Week 1: Clean the data. Run the dead deal purge (Phase 1). This is the foundation. Nothing else works if the pipeline is full of ghosts.
  2. Week 1: Create custom fields. Build all Organization, Person, and Deal custom fields. This is configuration only — fast to do, high impact.
  3. Week 2: Build the New Business pipeline with proper stages and probabilities. Migrate existing active deals into the new stages.
  4. Week 2: Deduplicate and classify orgs/contacts (Phase 2 cleanup). Populate Firm Type, Market Segment, and Relationship Type for all active records.
  5. Week 3: Build remaining pipelines (Assessment, Expansion & Renewal, Channel & Referral). Set up the labels and saved filters.
  6. Week 3: Set up automations. Start with the activity creation automations and stale deal alerts — these enforce behavior immediately.
  7. Week 4: Build dashboards. Start with Pipeline Health and Sales Activity. Add the others over time as data populates the new fields.
  8. Ongoing: Enforce data discipline. The configuration is only as good as the data that goes into it. Weekly pipeline reviews are non-negotiable.
